## 2. 用户故事地图

### 2.1 Happy Path

**路径A：工程师接单变现（核心用户流）**

```
[AI独立开发者]
  │
  ├─ 注册（邮箱/手机号）
  │   └─ 自动创建Privy MPC钱包（无感上链）
  │
  ├─ 完善资料（技能标签/学校/公司/GitHub）
  │   └─ 申请VERIFIED认证（GitHub 500+ Stars）
  │
  ├─ 浏览任务池
  │   └─ 筛选"Agent开发"/ 最高悬赏排序
  │
  ├─ 查看任务详情
  │   └─ Markdown需求描述 + $500 USDC悬赏 + 链上Escrow已锁定
  │
  ├─ 申请认领（填写方案说明）
  │   └─ 发布方审核 → 确认认领
  │
  ├─ 开发执行 → 提交交付（GitHub链接 + 文档）
  │
  ├─ 发布方验收确认
  │   └─ Escrow自动释放: 90% → 工程师钱包 / 10% → 平台
  │
  └─ 信誉SBT自动铸造
      └─ 个人主页显示信誉分 + 已完成任务
```

**路径B：企业发布任务找人（付费方流）**

```
[企业AI负责人/CAIO]
  │
  ├─ 注册 → 充值USDC到Privy钱包（从CEX转入）
  │
  ├─ 发布悬赏任务
  │   └─ 填写需求 + 存入$500 USDC到Escrow → 链上锁定
  │
  ├─ 收到认领申请
  │   └─ 查看申请者资料/信誉分/历史评价 → 选择最合适的
  │
  ├─ 等待交付
  │   └─ 收到交付物 → 确认验收（或要求修改）
  │
  └─ 资金自动释放 → 评价工程师
```

### 2.2 Edge Cases

| # | 场景 | 当前处理 | 建议 |
|---|------|---------|------|
| E1 | 工程师超时未交付 | 截止日+7天宽限→发布方可退款 | ✅ 合理，建议宽限期可配置 |
| E2 | 交付质量争议 | MVP阶段无仲裁机制 | MVP依赖发布方单方判断，v2.0加多签仲裁 |
| E3 | 发布方恶意拒绝验收 | MVP无防护 | **必须解决**——至少加"交付N天后自动验收"机制 |
| E4 | USDC价格波动 | Base上USDC稳定币 | ✅ 风险低，USDC锚定USD |
| E5 | 用户不会充值USDC | 充值引导页 | 需要详细的CEX→钱包教程 |
| E6 | Base链Gas异常飙升 | — | Base Gas极低，概率极低；加交易失败重试 |
| E7 | 工程师同时认领多个任务无法按时完成 | — | v1.1加并发认领上限（建议≤3个） |
| E8 | 发布方发布虚假任务 | — | MVP后加任务审核+举报机制 |

### 2.3 ⚠️ 关键Edge Case补充

**E3是MVP必须解决的：** 如果发布方可以无限期拒绝验收，工程师的权益无法保障。建议：

```
交付后72小时 → 系统自动提醒发布方验收
交付后7天 → 发布方未操作 → 自动验收（资金释放）
发布方可在7天内"要求修改"，重置计时
```

### 5.2 核心差异化壁垒

**壁垒1: 链上Escrow — 解决自由职业最大痛点**
- Upwork/猪八戒的中心化托管：平台可冻结、跑路、扣款
- 我们的链上Escrow：代码即法律，资金流向透明，无人可挪用

**壁垒2: SBT信誉 — 不可篡改的实战履历**
- 传统平台评价可被删除、刷单、购买
- SBT铸造在Base链上，不可转让、不可篡改、永久可查

**壁垒3: AI精英垂直 — 精准高密度匹配**
- 不做"所有行业的自由职业"，只做AI超级个体
- 社区内技能标签密度高，匹配效率远高于综合平台

**壁垒4: 无感上链 — Privy MPC钱包**
- Braintrust等Web3平台要求用户有钱包知识
- 我们注册即有钱包，用户零区块链知识即可使用

### 5.3 竞争风险

| 风险 | 概率 | 应对 |
|------|------|------|
| Upwork加AI垂直分类 | 中 | 链上信誉是迁移壁垒 |
| Braintrust扩展中文市场 | 低 | 先发优势+本地化运营 |
| Boss直聘做自由职业功能 | 中 | 定位不同（他们做雇佣，我们做悬赏） |
| 新的Web3竞品出现 | 中 | 快速上线+社区积累 |

## 6. MVP定义（4-6周开发量）

### 6.1 MVP一句话

**完成一条任务交易闭环（发布→接单→Escrow托管→交付→验收→链上结算）+ 精英浏览 + 链上信誉。**

### 6.2 MVP核心流程验收

| # | 场景 | 验收标准 |
|---|------|---------|
| 1 | 新用户注册 | 邮箱注册 → Privy钱包自动创建 → 完善资料 ≤3分钟 |
| 2 | 发布任务 | 填写信息 → 存入$100 USDC → Escrow锁定 → BaseScan可见 |
| 3 | 工程师认领 | VERIFIED用户 → 申请认领 → 发布方选中 → 合约记录executor |
| 4 | 交付验收 | 提交交付 → 确认验收 → 90% USDC到工程师 → 10%到平台 |
| 5 | 信誉铸造 | 任务完成后 → SBT自动铸造 → 个人主页显示信誉分 |
| 6 | 超时退款 | 超过截止日+7天 → 发布方退款 → 资金原路返回 |
| 7 | 无Web3体验 | 非Web3用户完成全流程无需理解区块链概念 |

### 6.3 MVP明确不做

- ❌ 兴趣小组/动态流/社交Feed
- ❌ 课程/学习路径/技能评估
- ❌ 黑客松/CAIO岗位
- ❌ 私信IM
- ❌ 法币支付
- ❌ NFT徽章/多签仲裁
- ❌ 积分等级体系

## 7. 产品风险（用户不买账的3个原因+应对）

### 风险1："搞什么区块链？我就是想接个活"——Web3抵触心理

**概率：** 40% | **影响：** 致命

**根因：** AI工程师群体对Web3有好感度分化，部分人直接因为"又要搞钱包"而流失。

**应对：**
- **无感上链是关键** — Privy MPC钱包，注册即有钱包，全程不提"区块链"
- **价值话术重写** — 不说"链上Escrow"，说"资金安全托管，平台无法挪用你的钱"
- **双轨体验** — MVP同时支持Privy钱包+MetaMask直连，满足不同用户
- **5人可用性测试** — 找5个非Web3的AI工程师，验证全流程是否"无感知"
- **v1.1考虑法币通道** — 微信支付/支付宝充值USDC，进一步降低门槛

### 风险2："平台上没几个任务，接不到活"——冷启动鸡生蛋

**概率：** 35% | **影响：** 致命

**根因：** 没任务→工程师不来→没工程师→企业不发布→死循环。

**应对：**
- **OPENCAIO官方发布50个种子任务**（预算¥50万）— 上线前就有活
- **种子用户定向邀请** — 清北/中科院AI方向KOL首批入驻
- **首月0佣金** — 降低双方试错成本
- **从社群切入** — OPENCAIO社群200名AI精英定向邀请，保证初始密度
- **质量>数量** — 20个高质量任务 > 200个低质量任务

### 风险3："我的$500被锁了，发布方不验收"——信任崩塌

**概率：** 20% | **影响：** 极高

**根因：** MVP阶段缺乏仲裁机制，发布方恶意拖延或不验收，工程师权益受损。

**应对：**
- **自动验收机制（必须加）** — 交付后7天未操作→自动验收→资金释放
- **平台介入** — MVP阶段人工处理争议（虽然不scale，但保证初始体验）
- **VERIFIED双向门槛** — 发布方也需要认证，降低恶意行为概率
- **v2.0加多签仲裁** — 5人仲裁委员会3/5投票决定
- **单笔上限$10,000** — MVP控制风险敞口

## 8. 北极星指标

### 核心指标

**北极星 = 月完成任务数（Completed Bounties per Month）**

> 完成任务 = 全流程闭环（发布→认领→交付→验收→结算）的任务数

### 为什么是这个指标？

| 候选指标 | 问题 |
|---------|------|
| 注册用户数 | ❌ 虚荣指标，注册≠使用 |
| 任务发布数 | ❌ 发布≠完成，可能有大量僵尸任务 |
| GMV（总悬赏金额） | ❌ 金额≠质量，可能少数大任务撑起GMV |
| **月完成任务数** | ✅ 直接反映平台价值——帮人赚到了钱 |
| 链上SBT铸造数 | ❌ 派生指标，完成任务→SBT铸造 |

### 支撑指标体系

| 类别 | 指标 | 目标（MVP后1月） |
|------|------|----------------|
| **获客** | 注册→VERIFIED认证转化率 | ≥30% |
| **激活** | 注册→首次浏览任务 | ≥60% |
| **核心** | 任务发布→完成率 | ≥50% |
| **核心** | 月完成任务数 | ≥20 |
| **留存** | 工程师W4留存 | ≥30% |
| **收入** | 平台佣金收入 | ≥$1,000 |
| **推荐** | VERIFIED用户推荐新用户率 | ≥10% |

## 附：评审总结

### ✅ 做对了的

1. **链上Escrow**是真正的差异化——解决自由职业信任痛点
2. **Privy无感上链**思路正确——降低用户门槛
3. **悬赏制而非竞价制**——保护工程师利益，避免Upwork式价格战
4. **SBT信誉系统**——构建长期护城河
5. **H5移动优先**——快速验证，不陷在小程序审核里
6. **Base链选择**——Gas极低+USDC原生+Coinbase背书

### ⚠️ 需要调整的

1. **必须加自动验收机制**（交付7天后）——否则工程师权益无保障
2. **冷启动策略要更具体**——50个种子任务之外，还需要10个KOL入驻
3. **成长Tab在MVP是空壳**——建议MVP隐藏成长Tab，只保留3个Tab（发现/任务/我的）
4. **VERIFIED标准可以更宽**——MVP阶段降低认证门槛，扩大可接单用户池
5. **USDC充值教程必须极致简化**——这是最大使用障碍

### 🔑 关键决策建议

1. **先做海外华人AI社区冷启动**——避免中国大陆加密监管风险
2. **MVP砍掉成长Tab**——避免用户看到空白页面产生负面印象
3. **种子任务预算提升到¥100万**——50个任务太少，至少100个才有"市场感"
4. **合约上线前必须做专业审计**——资金安全是生死线
