# 蓝血菁英 (Blue Blood Elite) — 产品需求说明书 v2.0

> **产品定位：** AI超级个体的精英社交 + 技能变现 + 链上信任平台
> **文档版本：** v2.0 (MVP聚焦版)
> **更新日期：** 2026-04-02
> **负责人：** OPENCAIO CPO

---

## 一、产品概述

### 1.1 一句话描述

蓝血菁英 = AI超级个体的 **任务悬赏平台** + **精英社交网络** + **链上信誉系统**。

### 1.2 核心价值主张

| 谁 | 痛点 | 我们的方案 |
|----|------|----------|
| AI工程师 | 技术强但缺乏变现渠道和高质量人脉 | 任务池悬赏 + 精英社交 + 链上信誉 |
| 企业/CAIO | 找不到靠谱的AI人才，资金托管不放心 | 发布任务 + Escrow托管 + 验收结算 |
| AI学习者 | 缺乏实战经验和行业认可 | 课程学习 + 任务练手 + 信誉积累 |

### 1.3 差异化竞争

1. **Escrow链上托管** — 资金安全透明，解决自由职业最大信任痛点
2. **链上信誉SBT** — 不可篡改的实战履历，形成真正的"区块链简历"
3. **AI精英垂直** — 不做泛招聘，只做AI超级个体的精准匹配
4. **无感上链** — Privy MPC钱包，用户无需懂区块链

## 二、MVP范围定义 (v1.0)

### 2.1 MVP核心原则

**只做一条完整链路：** 任务发布 → 接单 → Escrow托管 → 交付验收 → 链上结算。

MVP不做：社交推荐、课程系统、黑客松、岗位对接、VIP会员。

### 2.2 功能范围

| 模块 | MVP包含 | MVP不包含（v2.0+） |
|------|---------|-------------------|
| 用户系统 | 注册/登录、个人资料、VERIFIED认证 | 积分等级体系、成长飞轮 |
| 社交 | 精英列表浏览、详情页、基础搜索 | 智能匹配AI推荐、兴趣小组 |
| 任务池 | 发布/浏览/认领/交付/验收 完整流程 | 里程碑分批释放、争议仲裁 |
| 智能合约 | Escrow托管合约 + 信誉SBT | NFT徽章、多签仲裁、里程碑 |
| 支付 | USDC(Base链) + Privy钱包 | 法币支付、微信/支付宝 |
| 前端 | H5 Web应用（移动优先） | 微信小程序、App |

### 2.3 MVP用户流程（主流程）

```
企业方 → 注册 → 发布任务(填信息+存入USDC) → 等待接单 → 审核接单人 → 验收 → 确认释放资金
工程师 → 注册 → 完善资料 → 浏览任务 → 认领申请 → 开发交付 → 收到USDC
```

## 三、功能需求详述（MVP）

### 3.1 用户系统

#### US-101: 注册与登录
> **作为** 一个新用户，**我想要** 用邮箱/手机号注册账号，**以便** 加入蓝血菁英平台。

- 注册方式：邮箱 + 密码；手机号 + 验证码（二选一）
- 注册时自动创建Privy MPC钱包（后台静默创建）
- 注册后引导完善个人资料（姓名、技术栈、学校/公司）

**验收标准：**
- [ ] 邮箱注册：收到验证邮件 → 点击链接 → 激活成功
- [ ] 手机注册：发送验证码 → 输入验证码 → 激活成功
- [ ] 注册成功后自动跳转到个人资料编辑页
- [ ] Privy钱包地址在后台生成，用户无需操作

#### US-102: 个人资料管理
> **作为** 注册用户，**我想要** 编辑个人资料（头像、技能标签、简介），**以便** 让其他人了解我的能力。

- 头像上传（支持裁剪）、昵称（必填）、一句话简介
- 技能标签（多选预设 + 自定义，最多10个）
- 学校、公司、技术方向（下拉选择）
- GitHub主页（选填，用于VERIFIED认证）

**验收标准：**
- [ ] 个人资料保存后立即在精英详情页生效
- [ ] 头像上传支持 jpg/png，≤2MB
- [ ] 技能标签可增删，显示在个人卡片上

#### US-103: VERIFIED认证
> **作为** 有实力的工程师，**我想要** 获得VERIFIED认证标识，**以便** 在平台上获得更多信任和接单机会。

MVP简化认证标准（满足任一）：
- GitHub 500+ Stars（链接验证）
- 上传大厂工牌/在职证明（人工审核，48h内）
- 完成2个平台任务且满意度≥4.0

**验收标准：**
- [ ] 提交认证后状态变为"审核中"
- [ ] 审核通过后个人卡片显示蓝色VERIFIED标识
- [ ] 审核拒绝显示原因，可重新提交

### 3.2 精英社交（简化版）

#### US-201: 精英列表浏览
> **作为** 企业方/工程师，**我想要** 浏览平台上所有精英用户，**以便** 寻找合作对象。

- 卡片列表：头像 + 姓名 + 技能标签 + VERIFIED标识
- 支持按技术方向筛选（下拉选择）
- 支持按姓名/技能搜索（文本搜索）
- 分页加载，每页20人

**验收标准：**
- [ ] 列表按最新注册时间排序
- [ ] 筛选后结果实时更新
- [ ] 搜索支持中文和英文（技术栈关键词）

#### US-202: 精英详情页
> **作为** 用户，**我想要** 查看某个精英的完整资料，**以便** 评估是否合作。

- 封面图 + 头像 + 基本信息
- 技能列表（标签式展示）
- 个人简介（富文本）
- 链上信誉分（如有）
- 已完成任务数和历史

**验收标准：**
- [ ] 点击列表中的用户卡片进入详情页
- [ ] 返回按钮正常工作
- [ ] 链上信誉分从合约读取实时数据

### 3.3 任务池（核心）⭐

#### US-301: 发布任务
> **作为** 企业方/CAIO，**我想要** 发布一个悬赏任务并存入奖金，**以便** 吸引精英来接单。

- 填写：标题、详细描述、技术栈标签、交付标准、截止日期、悬赏金额(USDC)
- 发布时：选择"链上托管"，通过Privy钱包存入USDC到Escrow合约
- 发布后：任务出现在任务池列表，状态为"Open"

**验收标准：**
- [ ] 必填字段校验：标题、描述、截止日期、悬赏金额
- [ ] 悬赏金额必须 > 0，且 ≤ 钱包余额
- [ ] USDC存入成功后，Escrow合约状态变为 Open
- [ ] 发布成功后任务出现在列表中，可在"我的发布"中查看
- [ ] Base链上可通过BaseScan查看到对应交易

#### US-302: 浏览任务
> **作为** 工程师，**我想要** 浏览所有悬赏任务，**以便** 找到适合我的项目。

- 任务卡片：标题 + 技术标签 + 悬赏金额 + 截止日期 + 发布方
- 筛选Pills：全部 / 本地化部署 / AI模型 / Agent开发 / Web3
- 点击进入任务详情页

**验收标准：**
- [ ] 任务列表按发布时间倒序
- [ ] 筛选标签点击后列表实时过滤
- [ ] 已被认领的任务显示"已接单"标记但仍可见

#### US-303: 任务详情页
> **作为** 工程师，**我想要** 查看任务的完整信息，**以便** 决定是否接单。

- 完整描述（Markdown渲染）
- 需求清单 / 交付标准
- 截止日期 + 悬赏金额
- 发布方信息（头像、名称、信誉分）
- 链上Escrow状态（已锁定金额、合约地址）
- "申请认领"按钮

**验收标准：**
- [ ] Markdown描述正确渲染（代码块、列表、粗体等）
- [ ] 链上Escrow状态实时显示（从合约读取）
- [ ] 已认领的任务显示执行者信息，"申请认领"变为"已接单"

#### US-304: 认领任务
> **作为** 工程师，**我想要** 申请认领一个任务，**以便** 开始工作并赚取悬赏。

- 点击"申请认领" → 填写简短申请说明（为什么适合我、计划方案）
- 发布方收到通知 → 在"我的发布"中查看申请列表
- 发布方选择一个工程师 → 任务状态变为 InProgress
- 认领后Escrow合约记录executor地址

**验收标准：**
- [ ] 只有VERIFIED用户可以申请认领
- [ ] 发布方可以看到所有申请者的资料和信誉分
- [ ] 确认认领后，合约状态从 Open → InProgress
- [ ] 双方收到通知（站内消息）

#### US-305: 交付与验收
> **作为** 工程师，**我想要** 提交交付物，**以便** 获得悬赏付款。

- 工程师点击"提交交付" → 填写交付说明 + 上传附件/链接
- 发布方收到通知 → 查看交付物 → "确认验收" 或 "要求修改"
- 确认验收 → Escrow合约自动释放USDC给工程师（扣除10%平台佣金）

**验收标准：**
- [ ] 交付物支持文本 + 链接（MVP不做文件上传，用外部链接）
- [ ] 确认验收后合约自动执行 `approve()`，资金释放
- [ ] 工程师钱包余额实时更新
- [ ] 交易完成后铸造信誉SBT给双方
- [ ] 发布方可"要求修改"（附修改意见），不触发资金释放

#### US-306: 任务超时与取消
> **作为** 发布方，**我想要** 在工程师超时未交付时取回资金，**以便** 避免资金被无限锁定。

- 超过截止日期 + 7天宽限期 → 发布方可申请退款
- 退款释放Escrow中的USDC回到发布方钱包
- 工程师信誉分下降

**验收标准：**
- [ ] 宽限期内不可退款
- [ ] 宽限期后发布方可一键退款
- [ ] 退款后合约状态变为 Cancelled
- [ ] 工程师信誉SBT记录一次"超时未交付"

### 3.4 链上信誉系统

#### US-401: 信誉SBT铸造
> **作为** 完成任务的工程师，**我想要** 获得链上信誉SBT，**以便** 积累不可篡改的实战履历。

- 每完成一个任务，自动铸造一枚信誉SBT（ERC-5192）
- SBT属性：任务ID、完成时间、满意度评分、发布方签名
- 信誉分 = 综合计算（完成任务数 × 权重 + 满意度均值 × 权重）
- 信誉等级：新手(0-30) / 可靠(31-60) / 精英(61-90) / 传说(91-100)

**验收标准：**
- [ ] 任务完成后30秒内SBT铸造成功
- [ ] SBT在BaseScan上可查看，不可转让
- [ ] 信誉分在个人主页实时显示
- [ ] 信誉分计算公式公开透明

#### US-402: 链上信息展示
> **作为** 用户，**我想要** 在个人主页看到链上信誉信息，**以便** 展示我的实力。

- 链上信誉分（大字显示）
- 已完成任务数
- BaseScan链接（查看链上详情）
- 信誉等级徽章

**验收标准：**
- [ ] 信誉数据从链上实时读取，非缓存
- [ ] 点击BaseScan链接可在新窗口打开对应地址

---

### 3.5 个人中心

#### US-501: 我的钱包
> **作为** 用户，**我想要** 查看我的USDC余额和交易记录，**以便** 管理我的资金。

- USDC余额（大字显示）
- 收支明细列表：时间、类型（存入/收入/支出）、金额、关联任务
- "充值"按钮（引导从CEX转入USDC到Privy钱包）
- "提现"按钮（引导从Privy钱包转到CEX）

**验收标准：**
- [ ] 余额与链上实际余额一致（允许±0.01误差）
- [ ] 交易记录显示最近50笔
- [ ] 充值/提现为引导说明，MVP不做链上兑换

#### US-502: 我的任务
> **作为** 用户，**我想要** 查看我发布的任务和认领的任务，**以便** 跟踪进度。

- Tab切换："我发布的" / "我认领的"
- 每个任务卡片显示：标题、状态、截止日期、金额
- 点击进入任务详情

**验收标准：**
- [ ] 正确按Tab分类显示
- [ ] 状态标签颜色区分：Open(蓝)、InProgress(绿)、Completed(灰)、Cancelled(红)

## 四、非功能需求

### 4.1 性能
- 页面首屏加载 < 3秒（H5）
- 任务列表滚动加载，无明显卡顿
- 链上交易确认等待时间显示进度（Base L2约2-5秒确认）

### 4.2 安全
- 用户认证：JWT + Refresh Token
- API接口：速率限制（60次/分钟/IP）
- 合约安全：OpenZeppelin标准库 + 多签管理
- 资金安全：Escrow合约经过审计（MVP前至少内部审计）

### 4.3 兼容性
- 移动端浏览器：iOS Safari、Android Chrome（最近2个主版本）
- 桌面浏览器：Chrome、Firefox、Safari、Edge（最近2个主版本）
- 最小宽度 320px，最大宽度自适应

### 4.4 合规（MVP最低要求）
- 用户协议 + 隐私政策页面
- 数据收集最小化原则
- 不做法币支付（USDC直接链上交易，降低合规复杂度）

## 五、数据模型（核心）

```
users:          id, email, phone, nickname, avatar, bio, school, company,
                direction, github_url, privy_wallet_address, is_verified,
                created_at, updated_at

skills:         id, user_id, skill_name, proficiency(0-100)

bounties:       id, publisher_id, title, description(markdown), category,
                tech_tags[], reward_usdc, deadline, delivery_standard,
                escrow_contract_address, status(enum),
                claimed_by, claimed_at, delivered_at, completed_at

applications:   id, bounty_id, applicant_id, message, status(pending/accepted/rejected)

deliveries:     id, bounty_id, content(markdown), links[], submitted_at

reputation:     id, user_id, onchain_score, tasks_completed, avg_satisfaction

transactions:   id, user_id, type(deposit/withdraw/escrow_lock/escrow_release/commission),
                amount_usdc, bounty_id, tx_hash, created_at
```
